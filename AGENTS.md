# 递归研究 Agent 指令

## 长期目标

本项目的长期目标是打造一个尽可能强的 Balatro 求解器：在可重复评估的基础上持续提升策略质量、通关稳定性和得分能力。

把本仓库作为一个可反复运行的研究工作流，用于改进和评估 Balatro 自动化 agent，并逐步把它推进到强 Balatro 求解器。每次运行都应保留项目记忆，收集有来源支撑的证据，细化开放问题，并只对研究记录做最小且有用的更新。

研究工作流服务于应用代码，但它不是重写项目的授权。除非当前运行目标明确要求实现代码，否则不要修改应用代码。

## 每次研究运行前必须阅读

开始运行前，阅读：

- `AGENTS.md`
- `README.md`
- `research/README.md`
- `research/memory.md`
- `research/sources.md`
- `research/questions.md`
- `research/findings.md`
- `research/decisions.md`
- `research/runs/` 中已有文件，尤其是最近一次运行日志

如果运行依赖应用行为，只检查与问题直接相关的代码、测试、示例或决策日志。

## 研究运行流程

1. 说明本次运行目标和重要假设。
2. 从 `research/questions.md` 中识别过期、未解决或高优先级问题。
3. 添加新发现前，先检查已有发现。
4. 尽量优先从一手来源收集证据。
5. 用简洁且有来源支撑的内容更新对应研究文件。
6. 创建或更新当天运行日志：`research/runs/YYYY-MM-DD.md`。
7. 如果运行失败或未达成通关目标，必须基于日志或直接观察总结失败原因，并说明证据来源。
8. 如果失败原因、策略理解、重要未知项或下一步假设发生变化，必须更新 `research/memory.md`；如果不更新，必须在最终输出中说明原因。
9. 为下一轮运行给出一个明确的成功率估计（百分比或区间），并说明估计依据。
10. 每一轮运行结束都必须创建一个 git commit；只提交本轮产生或修改的文件，避免混入无关工作区改动。如果因用户要求、权限或工作区冲突无法 commit，必须在最终输出中说明阻塞原因。
11. 按下面的固定最终输出格式收尾。

## 记忆更新规则

- 仅当项目目标、稳定事实、当前理解、工作假设或重要未知项发生实质变化时，才更新 `research/memory.md`。
- 记忆要简洁。它应总结持久的项目状态，而不是复制每次运行日志。
- 除非有新证据明确取代旧结论，否则保留历史结论。
- 替换或修订结论时，记录原因，并指向新的证据或决策。
- 每次修改记忆时，都更新 `Last updated` 字段。

## 来源与证据规则

- 优先使用一手来源：仓库代码、测试、日志、官方文档、上游项目文档、schema 和直接观察到的行为。
- 二手来源只用于背景、评论或线索，且需要在其他地方确认。
- 在 `research/sources.md` 中记录长期有效的来源位置。
- 在 `research/findings.md` 中记录具体证据，细节要足够以后复查。
- 包含来源名称、URL 或文件路径；对可能变化的网络来源，记录访问日期；同时标注置信度。
- 不要把没有引用来源的主张当作稳定事实。

## 信息分类规则

- 稳定事实：有充分证据支持、预计在项目或上游来源变化前保持成立的陈述。记录在 `research/memory.md`。
- 工作假设：对下一次运行有用、但仍需验证的判断。记录在 `research/memory.md`。
- 开放问题：需要调查或决策的未知项。记录在 `research/questions.md`。
- 新发现：本次运行得到的、有证据支撑的观察。记录在 `research/findings.md`，并在运行日志中摘要。
- 决策：会影响后续工作的方向或约束。记录在 `research/decisions.md`。

## 去重与历史保留规则

- 添加发现前，搜索 `research/findings.md` 和近期运行日志，确认是否已有相同主张。
- 如果主张已存在，只有在补充了有意义的新证据、置信度或影响时，才更新原条目。
- 不要用不同措辞重复添加同一发现。
- 不要仅因为发现或决策过时就删除旧记录。
- 对过时决策，在 `Superseded by` 中标明被什么取代，并新增一条更新后的决策。
- 当新发现改变了旧发现的解释时，保留两者并说明关系。

## 每次运行后的固定最终输出格式

每次研究运行结束后，按以下格式回复：

```text
运行总结：
- 目标：
- 已阅读文件：
- 已采取行动：
- 新发现：
- 失败原因：
- 记忆更新：
- 决策更新：
- 开放问题：
- 下次成功率估计：
- 建议的下一次运行：
- 变更文件：
- Commit：
```

如果某个类别没有变化，写 `无`。

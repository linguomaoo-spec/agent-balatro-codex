# 研究工作流

这个目录是 Balatro agent 项目的持久研究层。使用它在多次 Codex 研究运行之间保留上下文，同时避免不必要地修改应用代码。

## 文件

- `memory.md`：长期项目记忆、稳定事实、工作假设和重要未知项。
- `sources.md`：来源清单和证据质量规则。
- `questions.md`：按优先级组织的研究问题 backlog 和已解决问题。
- `findings.md`：每次运行中发现的、有证据支撑的观察。
- `decisions.md`：应指导未来研究或实现工作的决策。
- `run_prompt.md`：启动新一轮研究循环时可复用的提示词。
- `runs/`：按 `YYYY-MM-DD.md` 命名的单次运行日志。

## 工作原则

每次运行都应该让研究记录前进一小步：解决或细化一个问题，补充证据，更新记忆，或记录决策。除非研究结构本身已经误导后续工作，否则避免大范围重写。

### 新发现（New Findings）

- 持续学习论文建议把自我改进放在 replay、外部验证、回归评估和稳定性-可塑性权衡下运行；这与本项目现有日志和 Markdown 策略记忆体系高度匹配。
- 当前 `Runner.run` 返回 `game_over`，但 `EvalResult.score` 只给 `game_over_win` 加胜利奖励，胜利奖励路径可能无法在 live run 中触发。

### 提升改造计划（Improvement Plan）

1. 评估可信度优先：统一胜负状态，固定开发 seed、回归 seed 和 held-out seed，输出胜率、最高 ante、失败阶段、错误动作、现金、小丑牌数量、分数差距和运行成本。
2. 建立 replay 经验库：从 JSONL 日志提炼成功局、失败局和关键决策片段，按阶段、ante、deck、stake、小丑牌标签和失败类型索引，供子 agent 任务包按 top-k 注入。
3. 建立策略晋升门槛：任何新策略或 genome 候选都必须同时报告目标 seed 收益、回归 seed 退化、held-out 表现和成本；旧 seed 明显退化时不晋升。
4. 阶段化增强 agent：先修 hand 牌型覆盖和 shop 小丑牌分类，再推进 blind 跳盲和 booster 选择；每个阶段都先写 Markdown 策略、再用日志验证、最后转 Python 或 genome 权重。
5. 引入安全自我迭代循环：子 agent 从 replay 中提出候选策略，主 agent 用 BalatroBot 固定 seed 验证，失败案例回写 replay；禁止只根据模型自评或单次成功修改稳定策略。
6. 长期再考虑搜索或学习模型：在评估和 replay 稳定后，再引入 Monte Carlo rollout、轻量价值估计或阶段专家策略；避免在奖励信号不可靠时先扩大模型复杂度。

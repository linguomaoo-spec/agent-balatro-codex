# 2026-06-14 研究运行

## 目标

实现真实 BalatroBot checkpoint beam rollout、硬优先级状态价值、部分有界基因变异、per-seed Pareto archive 和分层 cohort 晋升，并尝试执行 live 硬验收。

## 重要假设

- 当前未提交的 v8 策略是 baseline，不回滚已有策略改动。
- heldout 只在 regression 冠军确定后运行，不参与调参。
- BalatroBot `save`/`load` 能恢复完整随机状态；该假设必须由真实 smoke 验证。

## 已采取行动

- 阅读项目研究记忆、来源、问题、发现、决策和近期运行日志，并以 v8 三 seed 结果为 baseline。
- 增加 `BalatroBotClient.save_checkpoint`/`load_checkpoint`、`CheckpointSearchPlanner`、`StateValue`、checkpoint 场景库和搜索 JSONL 摘要。
- 手牌搜索候选扩展为最多 4 个 play 与 2 个 discard；Boss/商店预算、各阶段 horizon 和无嵌套搜索均由 `config/search.json` 控制。
- Genome 增加有界基因、约 30% 部分变异、整数阈值和双 elite 均匀交叉；旧 JSON 自动补默认基因。
- Evolution 增加 `RunOutcome`、Pareto archive、逐 seed regression gate、场景→dev→regression→heldout 分层评估及对应 JSON 产物。
- 在真实 `SELECTING_HAND` 与 `SHOP` 状态完成 checkpoint 往返、相同分支复现和 Planner 根恢复 smoke。
- 启动默认搜索 AGENT1 live run；因实测吞吐过低，在 7 分钟、10 个动作后主动停止，未把不完整 run 解释为策略结果。

## 验证结果

- `python3 -m unittest discover -s tests -v`：155 个测试全部通过。
- `python3 -m compileall -q balatro_agent`：通过。
- BalatroBot health：通过。
- SHOP 同分支复现：通过；两次购买 `Gluttonous Joker` 的状态摘要一致，根状态恢复一致。
- SELECTING_HAND 同分支复现：通过；两次相同 play 后的得分、抽牌和剩余牌数一致，根状态恢复一致。
- Planner live smoke：手牌与商店均恢复根状态并输出分支价值。
- 完整 dev/regression/heldout：未完成。

## 失败原因

- 默认完整搜索成本超出本轮可执行预算：6 分支手牌决策约 35--85 秒，3 候选商店约 50 秒；按 v8 的 378 个 dev 动作估算完整三 seed 约需数小时，8×3 进化更长。
- 部分动作出现“已生效但 HTTP 响应关闭”；已把后续实现改为 `transport_warning`，避免误计为新增策略 error。
- 因 dev 未完成，不能评估“至少 1 胜、另外两个 seed 不低于 v8”；因此按门禁规则没有运行 regression 和 heldout。

## 下一轮成功率估计

完成搜索成本优化后，下一轮“跑完 checkpoint dev 并得到至少 1 胜”的成功率估计为 25%--40%。依据是 checkpoint 分支可复现且 AGENT1 已有固定路线胜局，但当前通用 v8 为 0 胜，搜索价值模型尚未经过完整 seed 验证，且运行成本会限制可评估候选数量。

## 建议的下一次运行

先对 18 个场景测量每阶段单位分支成本，加入动态预算或分支结果缓存，把完整 dev 控制在可重复运行的时长内；随后只跑 dev，达到至少 1 胜且 AGENT2/3 不低于 v8 后再解锁 regression。

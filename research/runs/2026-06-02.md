# 研究运行：2026-06-02

## 运行目标（Run Objective）

实现一个显式启动/停止的本地人工游玩 recorder，让用户玩小丑牌时可以记录 BalatroBot 状态变化，供后续复盘和策略研究使用。

## 重要假设（Assumptions）

- 本次允许修改应用代码，因为用户明确要求“做一个”本地 recorder。
- recorder 应只读游戏状态，不监听系统键盘、鼠标或屏幕，也不执行游戏动作。
- 最小可用版本优先复用现有 BalatroBot `gamestate` 集成和 JSONL 日志格式；精确点击/按键、OCR 或 mod 事件日志留作后续问题。
- 工作区已有未提交改动，本次不回滚既有内容，提交时只提交本轮 recorder 相关文件。

## 已采取行动（Actions Taken）

- 用 TDD 新增 `tests/test_recorder.py`，先确认缺少 `balatro_agent.recorder` 时测试失败。
- 新增 `balatro_agent/recorder.py`，实现只读状态 recorder：
  - 轮询 `BalatroBotClient.gamestate()`。
  - 默认只在状态哈希变化时写入 `human_state_snapshot` JSONL。
  - 每条记录包含时间戳、状态摘要、状态哈希、前一状态哈希、终局标记和可选原始状态。
  - 默认遇到 `GAME_OVER` 后停止并返回 `game_over_win`、`game_over_loss` 或 `game_over`。
- 新增 `record` CLI。
- 新增 `scripts/record-human-start.sh` 和 `scripts/record-human-stop.sh`，提供显式后台启动/停止入口。
- 更新 `README.md`，记录人工游玩 recorder 用法和边界。
- 更新 `research/sources.md`、`research/findings.md`、`research/questions.md`、`research/decisions.md` 和 `research/memory.md`。

## 新发现（New Findings）

- 本仓库此前没有独立的人类游玩 recorder；现有日志主要来自 agent 决策和固定 seed 评估。
- BalatroBot 状态轮询可以作为低侵入的一手证据采集入口，但不能直接还原系统级点击或按键。

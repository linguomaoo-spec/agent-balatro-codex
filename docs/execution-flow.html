小丑牌通关执行流程

核心策略：从灵活探索到牌型专精 — 前期不设固定牌型，随小丑牌积累逐步锁定目标，后期集中所有资源（Joker、塔罗、星球、弃牌）打造最强牌型完成通关。

⭐ 核心策略

渐进式牌型专精 · 三段战略弧线

这是通关的核心方法论：Agent 不在一开始就固定出牌牌型，而是根据游戏进程中拿到的小丑牌、塔罗牌和星球牌，逐步收敛到一个最优牌型方向，然后用弃牌和商店决策全力打造该牌型。

STAGE 1

灵活探索

Ante 1 – 2
🎯 不设固定牌型，有什么打什么
  • 优先打出最高分的天然牌型（顺子/同花优先）
  • 观察前几张 Joker 的类型和触发条件
  • 商店阶段优先买通用型 Joker（不绑定特定牌型）
  • 塔罗牌优先转换为金钱或通用增强牌
  • 星球牌先不用，观望牌型方向
  • 不在此阶段投入大量资源重掷商店
STAGE 2

锁定方向

Ante 3 – 5
🎯 根据 Joker 组合锁定目标牌型
  • 根据已有 Joker 的触发条件确定主攻牌型
  • 商店只买与目标牌型协同的 Joker
  • 塔罗牌开始改造手牌：删杂牌、增强目标花色/点数
  • 星球牌集中升级目标牌型等级
  • 卖掉不再符合方向的早期 Joker
  • 弃牌开始为目标牌型服务：保留关键花色/点数
STAGE 3

全力打造

Ante 6 – 8
🎯 所有资源服务一个牌型
  • 只打目标牌型，不打任何其他牌型
  • 弃牌全力雕塑手牌：追目标花色/点数/对子
  • 星球牌堆满目标牌型等级
  • 塔罗牌全力增强/复制目标牌型关键牌
  • Joker 槽位按 Chip → +Mult → ×Mult 排序
  • 保留跨 Boss 现金，Boss 前不浪费重掷
  • ×Mult Joker 优先于 +Mult，成长型优先于固定型
牌型方向信号

当出现以下信号时启动牌型锁定：
· 拿到 2+ 张触发同一牌型的 Joker（如 Flush 的 Sinful Joker 系列）
· 拿到牌型专属成长 Joker（如 Pair 的 Supernova）
· 拿到强 X 倍率 Joker 且可匹配某个牌型
· 塔罗牌/星球牌集中指向某个方向

锁定后的连锁反应

一旦锁定目标牌型，所有 Agent 的决策权重都会偏移：
· HandAgent：目标牌型得分 ×2.0，其他牌型降权
· ShopAgent：匹配 Joker 升权，不匹配 Joker 降权
· ConsumableAgent：目标星球牌优先，塔罗牌服务目标花色
· EconomyAgent：为目标星球牌保留更多现金

反悔机制

以下情况可以改变目标牌型：
· 拿到更强的 X 倍率 Joker 且适配不同牌型
· 当前牌型星球等级已达瓶颈（如商店不再出现）
· Boss Blind 直接克制当前牌型（如 The Mouth 限制单牌型）
· 商店连续出现大量另一牌型的强力 Joker

弃牌雕塑策略

锁定牌型后的弃牌目标：
· 同花：弃掉非目标花色的所有牌 → 追同花
· 顺子：保留连续点数段 → 弃两端孤牌
· 对子/三条/四条：弃单张 → 保留成对点数
· Full House：追三条 + 对子组合
· 所有牌型：优先保留增强牌（Steel/Glass/Lucky）

01 · 入口

如何启动通关流程

所有流程需要在 BalatroBot 运行且游戏已启动到主菜单（MENU）状态时执行。

前置条件

1. 启动游戏 & 健康检查

通过 Lovely 注入加载 BalatroBot mod，确保 JSON-RPC 服务可达。

# 启动 Balatro + Lovely + BalatroBot sh "/Users/suriness/Library/Application Support/Steam/steamapps/common/Balatro/run_lovely_macos.sh" # 健康检查（绕过本地代理） NO_PROXY=127.0.0.1,localhost \ python3 -m balatro_agent --timeout 3 doctor
基础通关

2. 单局自动循环

最简通关入口：Runner 自动感知阶段、调用 Agent 决策、执行动作、记录日志，最多 500 步。

# 单局自动循环（无搜索） NO_PROXY=127.0.0.1,localhost \ python3 -m balatro_agent run \ --max-steps 500 \ --log runs/decisions.jsonl
带搜索增强

3. Checkpoint Beam 搜索模式

在关键决策点（商店/手牌）进行多分支前瞻搜索，通过 save/load 实现真实回滚，选取最优分支。

NO_PROXY=127.0.0.1,localhost \ python3 -m balatro_agent run \ --search \ --search-config config/search.json \ --max-steps 500 \ --log runs/search-decisions.jsonl
批量评估

4. 固定 Seed 多局评估

在 dev/regression/heldout 三组固定 seed 上评估 genome 表现，生成可复现的对比基准。

NO_PROXY=127.0.0.1,localhost \ python3 -m balatro_agent --timeout 15 eval \ --deck RED --stake WHITE \ --seed-config config/eval-seeds.json \ --cohort dev \ --max-steps 500 \ --log-dir runs/eval/live-YYYYMMDD-HHMMSS
基因组进化

5. 分层 Checkpoint 进化

先收集 checkpoint 场景 → 每代 top-3 进 dev → top-2 进 regression → 冠军跑 heldout。

NO_PROXY=127.0.0.1,localhost \ python3 -m balatro_agent --genome config/default-genome.json evolve \ --deck RED --stake WHITE \ --search \ --seed-config config/eval-seeds.json \ --generations 3 --population 8 \ --output-dir runs/evolution

03 · 手牌决策

HandAgent · 牌型评估 + 渐进式锁定 + 弃牌雕塑

在手牌阶段，HandAgent 评估所有可能的牌型组合。牌型权重不再静态：已锁定目标牌型后，目标牌型得分大幅提升，非目标牌型降权，弃牌策略从「丢散牌」升级为「主动雕塑目标牌型」。

🔬 牌型评分公式（牌型专精加权）
hand_score = base_score × genome.weight(hand_type) × commitment_multiplier(hand_type) + chip_contrib + mult_contrib + joker_synergy + enhancement_bonus - boss_penalty # commitment_multiplier = 2.0 当 hand_type == 已锁定目标牌型 # commitment_multiplier = 0.5 当 hand_type 与目标牌型不兼容且 ante ≥ 4 # 未锁定阶段（ante ≤ 2）= 1.0，所有牌型平等竞争
🎯

弃牌雕塑策略

不再是简单的"丢分数最低的散牌"。锁定牌型后，弃牌变成主动雕塑：同花方向弃非目标花色；顺子方向保留连续段弃孤牌；对子方向追成对点数弃单张。每次弃牌都是一步向目标牌型收敛。

⚡

Hanging Chad 重排

拥有 Hanging Chad（重触发第一张牌）时，自动把增强牌（Glass/Steel/Lucky/Mult）或高筹码牌排到出牌列表第一位，最大化首次触发收益。此逻辑与牌型专精无关，始终生效。

🛡️

Boss 感知

The Mouth 限单牌型 → 自动使用锁定牌型；The Needle 只能打一手 → 弃牌全力追目标；The Hook 弃牌减半 → 更早雕塑；The Wall/Violet Vessel → 更激进追高牌型而非安全 Pair。

04 · 商店决策

ShopAgent · 牌型驱动的购买/出售/重掷

商店阶段所有决策都应该围绕已锁定的目标牌型。Joker 不再按通用强度评分，而是按与目标牌型的协同度评分。塔罗牌和星球牌也以目标牌型为第一优先级。

🔬 牌型驱动的 Joker 评分
buy_score = joker_strength(item) × genome.weight("buy_joker") × hand_type_synergy(item, committed_hand) + boss_aware_adjust # hand_type_synergy 核心逻辑： # - 已锁定 Flush → Sinful Joker 系列（Wily/Gluttonous等）得分 ×3.0 # - 已锁定 Pair → Supernova/Business Card 得分 ×3.0 # - 已锁定 Straight → Runner/Shortcut 得分 ×3.0 # - 通用 Joker（Blueprint/Brainstorm）始终高分 # - 与目标牌型冲突的 Joker 直接大幅降权甚至排除
🔬 消耗品选择（塔罗牌 & 星球牌）
# 星球牌优先级： 买星球牌 = planet == committed_hand_type ? HIGH_PRIORITY : LOW_PRIORITY（除非 ante ≤ 2 探索期） # 塔罗牌使用策略（已锁定牌型后）： # - Flush：优先转换花色（Lovers/World/Sun）→ 统一花色 # - Pair/Three/Four：优先 Death → 复制高价值点数 # - Straight：优先 Strength → 调整点数连续性 # - 所有牌型：Hanged Man → 删除与目标牌型无关的牌 # - 通用：Hermit/Temperance → 经济保障；Fool → 复用最佳塔罗
🛒

牌型协同购买

遍历商店 Joker 时，核心评分维度从「通用强度」变为「与目标牌型的协同度」。窄条件 Joker 如果正好匹配目标牌型，反而大幅升权而非降权。

💸

跨 Boss 经济

保留足够的跨 Boss 现金（参考 AGENT1 胜局的 68/63 金下限），确保 Boss 后能立即购买关键 Joker 或星球牌。Boss 前不浪费重掷除非有明确替换目标。

🔀

Joker 排序 + 替换

已锁定牌型后，JokerOrderAgent 更激进地排列 Chip→+Mult→×Mult。卖掉与目标牌型无关的旧 Joker 时，替换门槛降低（更容易接受替换）。

05 · 路线示例

典型牌型专精路线

以下是三种最常见的牌型专精路线，展示从灵活探索到全力打造的完整演进。

🌸 同花路线

Flush → 统一花色

信号：拿到 2+ 张 Sinful Joker 系列（Wily/Gluttonous/Greedy/Lustful/Wrath）或 Ancient Joker。
锁定后：弃牌统一追单一花色；塔罗牌转换花色；星球牌只买 Jupiter；优先拿 Smeared Joker。
上限：高基础分 + 容易成型，但单一花色可能被 Boss 克制。

👫 对子路线

Pair / High Card → 小牌型爆发

信号：拿到 Supernova / Green Joker / Ride the Bus / Burnt Joker 等成长型 +Mult Joker。
锁定后：星球牌堆 Pluto/Mercury；塔罗牌增强/复制特定点数；弃牌追对子或多张同点数。
上限：出牌次数多、成长空间大，但需要强力 X 倍率来源补终局。

🔢 顺子路线

Straight → 连续点数

信号：拿到 Runner / Shortcut / Four Fingers / The Order。
锁定后：星球牌只买 Saturn；塔罗牌 Strength 调整点数；Hanged Man 删边缘牌；弃牌保留连续段。
上限：基础分高但成型难度大，Shortcut 和 Four Fingers 是关键。

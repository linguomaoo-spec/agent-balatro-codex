# 小丑牌策略记忆系统

这个目录保存可以反复使用的小丑牌策略记忆。策略记忆不是一次性的笔记，而是
后续研究、子 agent 任务、日志分析和策略实现的输入。

## 使用方式

- 先从 `strategy/index.md` 找到相关阶段、局型或问题。
- 新增策略前，先搜索现有策略和 `research/findings.md`，避免重复记录同一主张。
- 每条稳定策略都要能指向证据：本地日志、测试、代码行为、上游文档或明确的运行观察。
- 没有证据的判断写入“待验证问题”或“工作假设”，不要写成结论。
- 策略足够稳定后，再考虑是否把它转成 Python 规则或 genome 权重。

## 文件结构

- `index.md`：总索引和当前最高优先级记忆入口。
- `phases/hand.md`：出牌、弃牌和保留手牌策略。
- `phases/shop.md`：商店购买、重掷和离开商店策略。
- `phases/blind.md`：选盲、跳盲和 Boss blind 应对策略。
- `jokers/README.md`：小丑牌协同、优先级和反例。
- `runs/README.md`：把评估日志提炼为策略记忆的规则。

## 固定条目结构

每个策略条目优先使用这个结构：

```text
### 策略名称

- 适用局面：
- 决策规则：
- 证据来源：
- 失败案例：
- 待验证问题：
```

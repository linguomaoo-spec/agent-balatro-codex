# 手牌阶段策略记忆

## 范围

记录 `SELECTING_HAND` 阶段的出牌、弃牌和保留策略。当前 Python baseline 只解析
点数分组和高牌，后续策略应先在这里沉淀，再决定是否进入代码。

### 基线：优先可解释的有效出牌

- 适用局面：没有经过日志验证的更强规则时。
- 决策规则：优先选择已有代码能稳定识别的对子、三条、四条或高点数牌组，避免产生非法动作。
- 证据来源：`balatro_agent/agents.py` 中 `HandAgent._best_play_indices`；`tests/test_orchestrator.py` 覆盖对子出牌。
- 失败案例：尚未系统分析同花、顺子、葫芦等牌型被错过的频率。
- 待验证问题：当前失败 run 中，手牌阶段主要损失来自牌型识别不足、弃牌时机错误，还是缺少小丑牌协同判断？

### 工作假设：弃牌应服务于下一手收益

- 适用局面：仍有弃牌次数，且当前出牌不容易达到盲注要求。
- 决策规则：优先弃掉低价值、低协同、不会组成目标牌型的牌；保留对子、潜在顺子、潜在同花和被小丑牌强化的牌。
- 证据来源：当前仅为策略假设；需要从 `runs/*.jsonl` 或评估日志中验证。
- 失败案例：如果盲注分数压力很高，过度弃牌可能直接导致回合失败。
- 待验证问题：不同 ante、剩余手数和剩余弃牌数下，弃牌阈值应如何变化？

### 工作假设：优先追求顺子和同花

- 适用局面：手牌已有 4 张同花、4 张连续点数，或已经成型的顺子/同花，同时没有更强的小丑牌协同要求打单张、对子或三张以内牌。
- 决策规则：尽量把顺子、同花和顺子同花作为主要得分目标；弱对子、高牌不应轻易打掉潜在顺子/同花材料。已有成型顺子/同花时，除非葫芦、四条或关键小丑牌触发明显更强，否则优先直接出成型牌型。
- 证据来源：用户在 2026-06-01 提供的策略 know-how；仍需后续真实运行日志验证。
- 失败案例：盲注压力很高、剩余手数不足或没有弃牌次数时，强行追未成型顺子/同花可能错过当前可得分。
- 待验证问题：在当前 agent 中，顺子/同花优先级提高后是否能提升 round 15 之后的得分，同时不降低前期通过率？

### 待补充：小丑牌驱动的手牌选择

- 适用局面：持有改变牌型价值、花色价值、点数价值或出牌张数收益的小丑牌时。
- 决策规则：先记录具体小丑牌名称、触发条件、推荐牌型和反例，再考虑进入代码。
- 证据来源：待从本地运行日志和上游资料补充。
- 失败案例：未知。
- 待验证问题：哪些小丑牌最值得优先转成明确规则？
